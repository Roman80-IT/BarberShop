```html
<link rel="stylesheet" href="styles.css" type="text/css" media="all" />

<!-- Основний запис: -->
<link rel="stylesheet" href="styles.css" />
```

При підключенні CSS-файлу до HTML-документу найчастіше використовуються такі атрибути:

**rel:** вказує на відношення між поточним документом і пов'язаним ресурсом. Для підключення CSS використовують значення "stylesheet".

**href:** вказує на шлях до CSS-файлу.

**type:** вказує MIME-тип, який використовується для ресурсу. Для CSS це "text/css". Сьогодні цей атрибут зазвичай опускається, оскільки він за замовчуванням передбачається.

**media:** вказує, для яких пристроїв чи умов застосовуються стилі. Наприклад, "screen" для екранних пристроїв, "print" для принтерів або "all" для всіх пристроїв.

**title:** вказує назву стилів, яка може використовуватися при перемиканні між стилями.

**integrity:** використовується для забезпечення безпеки шляхом перевірки цілісності завантаженого ресурсу.

**crossorigin:** вказує, чи слід отримувати ресурс з підтримкою CORS.

### Приклади використання атрибутів

#### Підключення JavaScript з атрибутом `src`:

```html
<script src="script.js" type="text/javascript"></script>

<!-- Підключення JavaScript з атрибутом defer: -->
<script src="script.js" type="text/javascript" defer></script>

<!-- Підключення JavaScript з атрибутом async: -->
<script src="script.js" type="text/javascript" async></script>

<!-- Підключення JavaScript з атрибутами integrity і crossorigin: -->
<script
  src="script.js"
  type="text/javascript"
  integrity="sha384-..."
  crossorigin="anonymous"
></script>
```

## Яке значення атрибуту `type` вибрати - "text/javascript" чи "module"?

Вибір між значеннями атрибуту `type` для тега `<script>` — `"text/javascript"` і `"module"` — залежить від типу JavaScript-коду, який ви використовуєте.

### Порівняння `text/javascript` і `module`

| Значення атрибута `type` | Опис                                                                                                                   | Що вибрати в вашій ситуації                                                         |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| `text/javascript`        | Стандартний MIME-тип для звичайного JavaScript-коду. Це значення використовується для класичних скриптів.              | **Виберіть, якщо ви використовуєте звичайний JavaScript-код, не підтримує модулі**. |
| `module`                 | Вказує, що JavaScript-файл є модулем ES6. Модулі мають свій власний контекст і можуть імплементувати експорт і імпорт. | **Виберіть, якщо ви використовуєте сучасний JavaScript-код з модулями (ES6+)**.     |

### Як вибрати між `text/javascript` і `module`

#### 1. Використовуйте `text/javascript`, якщо:

- Ваш код не використовує модулі.
- Ви використовуєте старий JavaScript-код.

**Приклад:**

```html
<script src="script.js" type="text/javascript"></script>
```

2. Використовуйте `module`, якщо:

Ваш код використовує ES6 модулі.
Ви пишете сучасний JavaScript-код з імпортом/експортом.

Приклад:

```html
<script src="script.js" type="module"></script>
```

### Основні відмінності між `text/javascript` та `module`

| Параметр              | `text/javascript`                                  | `module`                                |
| --------------------- | -------------------------------------------------- | --------------------------------------- |
| **Скрипти**           | Скрипти виконуються в глобальному контексті        | Скрипти мають свій власний контекст     |
| **Імпорти/Експорти**  | Не підтримує імпорт і експорт модулів              | Підтримує імпорт і експорт модулів      |
| **Порядок виконання** | Виконуються синхронно (відразу після завантаження) | Модулі завантажуються асинхронно        |
| **Залежності**        | Не може вказати залежності через `import`          | Можна вказати залежності через `import` |
| **Використання**      | Застарілий або базовий код                         | Сучасний код з модульною системою       |

### Рекомендації по використанню

- Використовуйте **`text/javascript`**, якщо ваш код не використовує модулі ES6 і просто виконує певну логіку на сторінці.

- Використовуйте **`module`**, якщо ваш код є частиною сучасного JavaScript-програми, де ви використовуєте імпорти і експорти між файлами.

### Резюме

| Значення `type`   | Опис                                                                            | Використання                     |
| ----------------- | ------------------------------------------------------------------------------- | -------------------------------- |
| `text/javascript` | Стандартний тип для JavaScript-коду без модулів                                 | Звичайні скрипти                 |
| `module`          | Використовується для модулів ES6, які можуть імплементувати `import` і `export` | Модулі ES6+, сучасний JavaScript |

## Що таке MIME-тип?

**MIME-тип** (Multipurpose Internet Mail Extensions type) — це стандартний спосіб опису типу даних, які передаються через Інтернет.

### Структура MIME-типу

MIME-тип має формат `type/subtype`, де:

- **`type`** — основний тип даних.
- **`subtype`** — підкатегорія типу даних.

### Приклад MIME-типів

| MIME-тип                            | Опис                                  |
| ----------------------------------- | ------------------------------------- |
| `text/html`                         | HTML-документ                         |
| `text/css`                          | CSS-файл                              |
| `text/plain`                        | Простий текст                         |
| `application/javascript`            | JavaScript-файл                       |
| `application/json`                  | JSON-дані                             |
| `image/jpeg`                        | Зображення JPEG                       |
| `image/png`                         | Зображення PNG                        |
| `audio/mpeg`                        | Аудіофайл MP3                         |
| `video/mp4`                         | Відеофайл MP4                         |
| `application/xml`                   | XML-документ                          |
| `application/pdf`                   | PDF-файл                              |
| `application/x-www-form-urlencoded` | Дані з форм, закодовані в URL-форматі |

### Важливість MIME-типів

1. **Ідентифікація Типу Даних:**  
   MIME-типи допомагають серверам і браузерам визначити тип вмісту, який вони отримують або відправляють.

2. **Обробка Даних:**  
   Браузери та інші клієнти використовують MIME-типи для вибору правильного способу обробки даних.

3. **Передача Даних:**  
   MIME-типи є частиною заголовків HTTP-запитів і відповідей. Вони вказують, які дані передаються і як їх потрібно обробляти.

### Як вказати MIME-тип у HTML

#### Атрибут `type` у `<script>`

```html
<script src="script.js" type="text/javascript"></script>
```
